Temos implementado o fluxo de Client Credentials, onde uma aplicação envia ao nosso oauth um conjunto de informações e recebe de volta um token de acesso.

Este fluxo deve ser utilizado apenas entre sistemas de backend que se comuniquem com o nosso gateway.

Fluxos com usuário final, em que o mesmo seja utilizado em aplicações frontend, deverão utilizar o fluxo especifico de troca de token descrito mais abaixo

## URL Base

**Homologação**: https://api-hml.valid.com

**Produção**: https://api.valid.com

### **Token Opaco**

Utilizamos de um Token Opaco em que, a partir dele, não é possível recuperar qualquer informação do mesmo. Pela vantagem de seu tamanho ser muito pequeno e seu consumo (sua validação) ser extremamente rápida.
